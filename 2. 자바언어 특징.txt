1. 자바 소개
    1) 1991년 썬 마이크로시스템즈 제임스 고슬링의 팀에서 시작되었다
    2) C/C++ 언어(포인터)가 주류였는데, 다양한 플랫폼마다 매번 다른 기계어로 컴파일을 해야한다
    3) 플랫폼의 독립적인 기능이 필요했다
    4) 자바가 가전제품, 휴대용 장치에 사용되는 소프트웨어 언어로 개발되었다
    5) 1996년 JDK(Java Development Kit) 1.0 공개되었다
        - JDK 1.1
        - J2SE 1.2(Java 2 Platform Standara Edition)
        - J2se 1.3
        - J2SE 1.4
        - J2SE 5.0
        - JAVA SE 6
        - JAVA SE 8 (2014) : LTS 버전의 시작
        ...
        - JAVA SE 19


2. 자바 프로그래밍의 특징
    1) 플랫폼에 영향을 받지 않으므로 다양한 환경에서 사용할 수 있다 (독립적이다)
            C 소스 코드(test.c) > 윈도우 컴파일러 > 윈도우 실행 파일 > 윈도우에서 실행
            C 소스 코드(test.c) > 리눅스 컴파일러 > 리눅스 실행 파일 > 리눅스에서 실행
            = C 소스 코드(test.Java) > 자바 컴파일러 > 바이트 코드(test class) > 윈도우 자바 가상머신, 리눅스 자바 가상머신, MacOS 자바 가상머신 > 윈도우, 리눅스, MacOS에서 실행
            MyApp
        JDK(JAVA Development Kit)
       ㄴ (JRE)            ㄴ (개발 도구) 
            ㄴ(JVM, API)                                        MyApp
        운영체제(windows, macos, liunx ...)                     Window
        하드웨어
    
        - 용어 정리
            - JDK ( Java Development Kit)
                - 자바에서 제공되는 개발용 라이브러리
            - JRE ( Java Runtime Environment)
                - 자바 프로그램이 실행되는 환경
            - JVM ( Java Virtual Machine)
                - 자바 가상 머신으로 프로그램이 실행되는 환경 JRE가 설치되어 있어야 한다
    
    2) 객체 지향 언어이기 때문에 유지보수가 쉽고 확장성이 좋다
        - cf. 절차 지향 프로그래밍 (procedural programming)
              객체 지향 프로그래밍 (Object oriented Programming)
    
    3) 프로그램이 안정적이다
    4) 오픈 소스 (풍부한 기능 제공)

3. 자바를 활용한 프로그래밍
    1) 웹 서버를 활용한 사이트 구축
    2) 안드로이드 프로그래밍
    3) 게임 프로그래밍

    3) 자료 구조 이해하고 구현하기
    4) 문제 해결하기
