1. 컴퓨터에서 데이터의 처리 방식
    1) 컴퓨터에서 숫자 데이터(2진수)를 표현하려면 전기 신호(0, 1) 이진수로 보내야한다
    2) 컴퓨터에서 정보를 처리하는 기본 단위는 바이트
    3) 1바이트로 양수만 표현하려면 ( 0 ~ 255 )까지 표현할 수 있다

2. 기본 자료형(primitive data type, 8개)
        
                정수형      문자형      실수형      논리형   
    
    1바이트     byte                               boolean
    2바이트     short       char
    4바이트     int                     float
    8바이트     long                    double

3. 정수 자료형
    바이트      자료형                 수의 범위
    1           byte          -2^7 ~ 2^7-1(-128 ~ 127)
    2           short         -2^15 ~ 2^15-1(-32,748 ~ -32,747)
    4           int           -2^31 ~ 2^31-1(-2억 ~ 2억)
    8           long          -2^63 ~ 2^63-1()

    1) byte
        - 1바이트 단위의 자료형 동영상, 음악 파일, 실행 파일의 자료를 처리할 때 사용한다
    2) short   
        - 2바이트 단위의 자료형 C/C++ 언어와 호환 시 사용한다
    3) int
        - 자바에서 사용하는 정수에 대한 기본 자료형
        - 가장 많이 쓰는 정수 자료형
        - 4바이트 단위의 자료형
        - 프로그램에서 사용하는 모든 숫자는 int로 저장된다
        - 32비트를 초과하는 숫자는 long 자료형으로 처리한다
    4) long
        - 8바이트 자료형
        - 숫자 뒤에 알파벳 L 또는 l을 long형임을 표시한다

4. 실수형(부동 소수점 방식, 고정 소수점 방식)
    0) 실수는 double 자료형이 기본이다
    1) 실수는 정수보다 정밀하기 때문에 정수와는 다른 방식으로 표현해야한다
    2) 실수 값 0.1
        1.0 * 10^-1
        ---   --  --
        가수  밀수 지수
    3) 지수부와 가수부로 표한한다
    4) 컴퓨터에서는 밑수를 2로 사용(정규화)
    5) 정규화 : 가수가 밑수보다 작은 한 자리까지 가수로 표현되는 것이다
        ex) 0.2 표현
            0.4 * 2^-1 정규화하면 1.6 * 2^-3  
    
    6) 자료형       크기
        float       4바이트
        double      8바이트 

5. 문자형
    1) 문자도 정수로 표현한다
        - 내부적으로 비트의 조합으로 표현이 된다
    2) 어떤 문자를 컴퓨터 내부에서 표현하기 위해서 특정 정수 값을 정의한다
        
        A(문자) > 65 (ecoding : 각 문자가 특정한 숫자 값(코드 값)으로 변환되는 것)
        A(문자) < 65 (decoding : 숫자에서 원래의 문자로 변한되는 것)  
        

    3) 문자세트 (charater set)
        - 문자를 위한 숫자 값(코드 값)들을 정해 놓은 세트
        - 각 문자를 숫자 얼마로 표현할 것인지 코드 값을 모아둔 것이다
        - ASCII code : 알파벳과 숫자, 특수 문자등을 1바이트에 표현하는 데에 사용하는 문자 세트 
        - euc-kr
        - 유니코드(Unicode)
            - 한글과 같은 복잡한 언어를 표현하기 위한 표준 인코딩이다
            - utf-8(유니코드) : 1바이트에서 4바이트까지 다양하게 문자를 표현할 수 있다
            - utf-16 : 2바이트로 문자를 표현

    4) 자바는 문자를 나타내기 위해서 전세계 표준인 UNICODE를 사용한다
    5) 자료형 이름          자료형 크기

       char                   2바이트
       
       char ch = 'A';

    6) 내부적으로 숫자로 표현되므로 숫자로 넣어도 문자가 출력될 수 있다

6. 논리형
    1) 자료형 이름              자료형 크기
        boolean                  1바이트

    2) true(참), false(거짓) 두 가지만 나타낸다
    3) 값이 존재하는지, 배열이 비었는지, 결과가 참인지 거짓인지 등을 표현한다

7. Local variable type inference
    1) 지역 변수 자료형 없이 사용하기
    2) 추론 가능한 변수에 대한 자료형을 선언하지 않는다
    3) 한번 선언하여 추론된 변수는 다른 타입의 값을 대입할 수 없다
    4) 지역 변수만 사용 가능하다
